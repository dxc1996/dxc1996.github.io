laravel-通知

3月 08, 2017 发布在 后端

当laravel自带的通知满足不了我们需求的时候的时候 我们就需要去自定义一个通知了

假设我们需要一个给dingding群推送消息

逻辑是这样的:

当实例化通知类的时候会先走Notification的via()方法在里面return要使用的发送频道 例如:return['mail']
然后会去channel文件中执行send方法，在send方法里面需要得到toMail()方法中传的参数而toMail()方法又要
去实例化Message所以一个通知需要有三个文件Notification、Channel、Message

第一步我们要生成这个通知
php artisan make:notification SendNotice
接下来我们需要去自定义这个通知了

1、在SendNotice要写via()和toDIngding()方法

public function via($notifiable)
{
    return [DingdingChannel::class, 'database'];
}
public function toDingding($notifiable)
{
    return (new DingdingMessage())
        ->content('你好');
}

2、定义需要的Channels叫DingdingChannel.php

注: 请求第三方接口的时候需要使用guzzlehttp/guzzle这个扩展

DingdingChannel.php的作用就是去发送这个通知 所以里面要写send这个方法

class DingdingChannel
{
    public function send($notifiable, Notification $notification)
    {
        $client = new Client();
        $message = $notification->toDingding(notifiable);
        $to = $notifiable->routeNotificationFor('dingding');
        $ret = $client->request('POST', $to, [
            'verify' => false,
            'json' => [
                'msgtype' => 'text',
                'text' => [
                    'content' => $message->content
                ]
            ]
        ]);
    }
}

3、定义Message叫做DIngdingMessage.php,上面toDingding()里面new的那个对象就是这里的Message

class DingdingMessage
{
    public $content;

    public function content($content)
    {
        $this->content = $content;
        return $this;
    }
}

4、通过官方文档和官方写的MailMessage，MaileChannel文件 我们发现我们可以通过在User.php这个model里面通过写

public function routeNotificationForDingding()
{
    return env('DINGDING_HOOK');
}

把需要的第三方接口url地址放到.env文件里 通过这样可以让DingdingChannels通过 $notifiable->routeNotificationFor(‘dingding’);获取url地址

假设我们需要给用户发送短信，用户的手机号我们也可以通过这种方式传给SmsChannel.php使用$notifiable->routeNotificationFor(‘sms’)获取

public function routeNotificationForSms()
{
    return $this->mobile;
}
